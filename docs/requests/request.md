## 3. 双端同步机制 (Sync Engine)

| # | 同步场景 | 期望行为 | 状态 |
|---|----------|----------|------|
| 3.1 | 教练发布训练 -> 队员端可见 | 教练点击"发布"后，队员端在 30s 内自动拉取到新计划 | ✅ 已验证 |
| 3.2 | 队员提交反馈 -> 教练端可见 | 队员提交周总结/日反馈后，教练端在 30s 内自动看到 | ✅ 已验证 |
| 3.3 | 教练打卡 -> 队员端日历更新 | 教练标记出勤后，队员端出勤日历自动反映 | ✅ 已验证 |
| 3.4 | 队员更新状态 -> 教练端监控面板更新 | 队员修改 Readiness/状态后，教练端 `SwimmerStatusPanel` 自动更新 | ✅ 已验证 |
| 3.5 | 教练回复评语 -> 队员端可见 | 教练写完评语后，队员端 CoachReplyPanel 每 15s 轮询，可看到最新批复 | ✅ 已验证 |
| 3.6 | 教练发起专项问卷 -> 队员端收到通知 | 教练发送问题后，队员端"反馈"Tab 出现待办红点 | ✅ 已验证 |
| 3.7 | 变动保护锁（Mutation Guard） | 本地修改后 15 秒内不被服务端旧数据覆盖 | ✅ 已验证 |

### 同步技术实现
- **机制**：30 秒轮询（`setInterval`），非 WebSocket
- **CoachReplyPanel 专项**：15 秒独立轮询，确保评语更快到达队员
- **防冲突**：15 秒 Mutation Guard（`recordMutation()`）
- **乐观更新**：所有写操作先更新本地 UI、再异步发送至服务器
- **回滚**：网络失败时回滚本地状态（出勤、队员添加等）
